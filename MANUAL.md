## 1. 系統概覽

### 1.1 系統架構

```
影片輸入
   ↓
pipeline（任務調度，`src/pipeline.py`）
   ↓
┌──────────────┐
│ 巨觀分析     │  → 隊形、熱區、互動密度、參與度
│ (Macro)      │
└──────┬───────┘
       ↓
┌──────────────┐
│ 微觀分析     │  → 同步度、穩定度、流暢度
│ (Micro)      │
└──────┬───────┘
       ↓
┌──────────────┐
│ 指標核查     │  → 紅/黃/綠燈評級
│ (Metrics)    │
└──────┬───────┘
       ↓
┌──────────────┐
│ 教育建議     │  → 教師回饋、家長文字
│ (Edu Advisor)│
└──────────────┘
       ↓
   完整報告
```

### 1.2 程式模組清單（詳見 `docs/agents/README.md`）

| 模組 | 功能說明 |
|-------|---------|
| `pipeline` | 任務總調度，串聯整個分析流程 |
| `macro_analytics` | 分析班級整體的隊形、空間使用、群體互動 |
| `micro_analytics` | 分析個別幼兒的節奏同步、穩定度、流暢度 |
| `metrics_checker` | 核查指標是否達標（紅/黃/綠燈） |
| `edu_advisor` | 生成教育建議與家長聯絡簿文字 |

## 2. 前置準備

### 2.1 影片拍攝建議

為了獲得最佳分析效果，建議拍攝時注意以下事項：

| 項目 | 建議 |
|-----|------|
| **拍攝角度** | 俯角或正面平視，避免嚴重遮蔽 |
| **解析度** | 至少 720p，越高越好 |
| **光線** | 充足且均勻，避免逆光或陰影過重 |
| **人數** | 建議 3-20 位幼兒 |
| **影片長度** | 建議 2-10 分鐘 |
| **檔案格式** | .mp4、.mov、.avi 皆可 |
| **老師入鏡** | 老師可入鏡，但系統以幼兒為主要分析對象 |

### 2.2 環境檢查清單

拍攝前確認：
- [ ] 教室空間足夠幼兒移動
- [ ] 拍攝範圍涵蓋主要活動區域
- [ ] 背景單純，減少無關物體干擾
- [ ] 音樂聲音清晰（系統需要偵測 BPM）

## 3. 基本操作

### 3.1 啟動完整分析

**方式**：上傳影片並說「分析」

```
用戶：傳送影片 + 「幫我分析這段影片」
系統：啟動 pipeline（完整管線）
      → 依序執行 Macro → Micro → Metrics → Edu Advisor
      → 產出完整分析報告
```

**輸出內容**：
- 基本資訊（時長、人數、解析度）
- 巨觀層分析（隊形變化、熱區分布、互動密度）
- 微觀層分析（同步度排名、穩定度、關注名單）
- 指標核查結果（紅/黃/綠燈）
- 教育建議與家長聯絡簿草稿

---

### 3.2 只做巨觀分析

**方式**：說「分析巨觀層」或「分析班級整體」

適用情境：只想快速確認班級整體狀況，不需要個別分析。

**輸出**：隊形結構、空間熱區圖、群體互動密度、參與度

---

### 3.3 只做微觀分析

**方式**：說「分析微觀層」或「分析個別孩子」

適用情境：老師已觀察到特定幼兒需要關注，進一步查看數據。

**輸出**：節奏同步誤差、抑制控制評估、動作流暢度排名

---

### 3.4 只檢查指標

**方式**：說「檢查指標」或「看看達標了嗎」

適用情境：已有分析數據，想確認指標是否達標。

**輸出**：各指標的紅/黃/綠燈狀態、綜合評分、需關注名單

---

### 3.5 生成教育建議與成長報告

**方式**：說「給我建議」、「寫家長聯絡簿」或「查看小明的成長趨勢」

適用情境：需要具備教育溫度的文字建議，或查看特定幼兒長期的進步軌跡。

**輸出**：
- 班級整體亮點回饋
- 個別進步見證與介入策略
- 家長聯絡簿溫馨草稿 (AI 生成)
- 月度/季度成長趨勢圖表 (個案分析)

## 4. 進階功能

### 4.1 針對特定時間區間分析
**方式**：說「分析 01:30 到 02:45 的片段」

### 4.2 只分析特定幼兒
**方式**：說「只看孩子 A」或「追蹤小明這學期的表現」

### 4.3 比較多次課程 (成長追蹤)
**方式**：上傳新影片後，說「比較這幾次課程」或「查看小明的成長軌跡」


---

### 4.4 調整分析靈敏度

如果分析結果與老師觀察有明顯落差，可以說「靈敏度調高」或「放寬標準」。

| 調整方向 | 適用情境 |
|---------|---------|
| 同步誤差門檻放寬 20ms | 老師認為同步度其實不錯，但系統判定過嚴 |
| 抑制控制位移門檻放寬 3cm | 孩子整體表現佳，但門檻過嚴造成過多紅燈 |
| 參與度門檻調低 10% | 教室空間大，孩子分散是正常現象 |

## 5. 報告解讀

### 5.1 巨觀層指標

| 指標 | 代表的意義 |
|-----|-----------|
| **隊形維持率** | 孩子服從指令、維持團體秩序的能力 |
| **熱區分布** | 教室空間是否被有效利用，孩子是否願意探索 |
| **互動密度** | 孩子之間的社交互動強度 |
| **參與度** | 整體課程的投入程度 |

### 5.2 微觀層指標

| 指標 | 代表的意義 |
|-----|-----------|
| **節奏同步度** | 與音樂的連結、節奏感、專注聆聽能力 |
| **抑制控制（停止後穩定度）** | 衝動控制、執行功能、專注切換能力 |
| **動作流暢度** | 肢體協調、大動作發展成熟度 |

### 5.3 指標燈號的意義

| 燈號 | 意思 | 行動建議 |
|-----|------|---------|
| 🟢 綠燈 | 表現優秀 | 維持當前教學策略 |
| 🟡 黃燈 | 良好但有進步空間 | 持續觀察，可適度調整 |
| 🔴 紅燈 | 明顯落後，需關注 | 設計針對性活動或個別介入 |

## 6. 常見問題

### Q1: 影片需要多長？
建議 2-10 分鐘。太短（<30 秒）數據不夠充分；太長（>30 分）建議分段處理。

### Q2: 可以分析即時影像嗎？
目前建議使用課後離線分析，確保微觀層分析準確度。

### Q3: 如何保護幼兒隱私？
系統僅使用代號（孩子 A/B/C），不涉及真實姓名。報告預設儲存在本地，不上傳雲端。

### Q4: 系統會取代老師的判斷嗎？
不會。所有分析結果僅供輔助參考，最終判斷權始終在教師。

### Q5: 如果燈號都是綠燈，是否代表一切都很好？
不一定。綠燈代表數據達標，但課堂教學仍有許多面向（如情感支持、創意表達）無法被數據捕捉。請以老師的專業觀察為主。

### Q6: 數據和老師觀察不一致時怎麼辦？
老師觀察優先。系統數據協助驗證與補充，但不應覆蓋老師的第一手觀察。可透過調整門檻或回報問題來持續優化系統。

## 7. 隱私與倫理

### 7.1 資料使用原則

1. **最小必要**：只收集分析所需的最少資料
2. **本地儲存**：分析結果預設存在本地，不上傳外部雲端
3. **去識別化**：所有報告使用代號，不使用真實姓名
4. **教師主控**：所有資料的最終控制權在帶班教師

### 7.2 對外溝通原則

| 情境 | 建議做法 |
|-----|---------|
| 與家長溝通 | 先經教師審核報告內容，用正向語言包裝 |
| 對外分享 | 僅分享趨勢性結論（如「全體同步度進步 15%」），不揭露個別幼兒數據 |
| 學術發表 | 須去除可識別資訊，並取得家長同意書 |

### 7.3 系統限制聲明

本系統所產出的任何報告、指標與建議：
- 均為 AI 輔助生成，僅供教學參考
- **不得**作為幼兒評量、獎懲或診斷的唯一依據
- 不得作為歧視或標籤化任何幼兒的工具
- 發現任何偏差或錯誤，請即時回饋以利修正
